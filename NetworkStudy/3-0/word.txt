■ ネットワーク層　…　通信プロトコルの機能や役割を構成する層

IP　…　IPアドレスを使用して送信元から宛先まで、データを配送する
　　　　インターネット層で使用されている主なプロトコル

IPヘッダ　…　ＩＰで扱うデータはＩＰパケットといい、
　　　　　　　そこに含まれるＩＰヘッダには住所であるＩＰアドレスが格納されている

ICMP　…　ネットワーク上での通信状況の確認や
　　　　　通信エラー時のメッセージの送信を主な用途とするプロトコル

ICMPメッセージ…　タイプフィールドの性質によって
　　　　　　　　　　QueryメッセージとErrorメッセージがある（確認時・緊急時）


IPアドレス　…　ネットワークの住所宛先

オクテット　…　IPアドレス３２ビットを８ビットずつに区切る。
　　　　　　　　この８ビットごとの単位


ネットワーク部　…　ネットワークを識別する
　
ホスト部　…　ホストを識別する（PCやルータなど）

クラス　…　昔は、ネットワーク部とホスト部の境界をアドレスの範囲で決めていた

プライベートIPアドレス　…　家庭や会社などの組織内のネットワークで使用するＩＰアドレス

グローバルIPアドレス　…　プライベートアドレスからネットワークへ通信できないため、ルータを通して変換して接続するためのＩＰアドレス
　　　　　　　　　　　　　→ＮＡＴという仕組み

ネットワークアドレス　…　ホスト部がすべて０のアドレス。ネットワーク自体を指す

ブロードキャストアドレス　…　ホスト部がすべて１のアドレス。同一ネットワーク上に接続されている
　　　　　　　　　　　　　　　すべてのホストにパケットを一斉送信するためのアドレス

ループバックアドレス　…　　その機器が起動していればＩＰ使うことができる。自分自身のこと。
　　　　　　　　　　　　　　例として、ＰＣⒶがループバックアドレスにつなげると、ＰＣⒶにくる。帰ってくる。

サブネット　…ネットワークを分割する
　　　　　　　→セキュリティの向上、ブロードキャストの制御、ＩＰアドレスの節約

サブネットマスク…ＩＰアドレスを、ネットワーク部とホスト部を識別するための数値
　
プレフィックス表記　…　例　～/２３

デフォルトゲートウェイ…　内部ネットワークと外部ネットワークをつなぐためのネットワーク機器。異なるネットワークの出入り口
（例：ルータ）

IPv6…ＩＰｖ４が枯渇してきたため、新しく作成されたプロトコル。ただ現状としてＩＰｖ４がいまだ主流
　　　　　→16進数。128ビットのため１６ビットずつ分ける

サブネットプレフィックス　…　IPv4でいうネットワーク部
インターフェイスID　…　IPⅴ4でいうホスト部
フィールド　…　IPv4でいうオクテット

ルータ…　異なるネットワーク間で橋渡しを行うには、中継役となる機器が必要。レイヤ３（ネットワーク層）で使われる

コンソール接続　…　初期設定で行われる接続。コンソールケーブル（ロールオーバーケーブル）を使って直接接続して設定を行う。

VTY接続　…　遠隔操作用のプロトコルを使用して機器にリモート接続を行い、設定や状態の確認を行う。
　　　　　　　VTYという仮想端末回線にTELNETやSSHなどのプロトコルを使用して接続する

AUX接続　…　モデムを通じて公衆回線（電話回線やISDN回線）を利用して機器にアクセスする方法

ルーティングテーブル…　ルータがパケットの転送先を判断するため参照するデータ記録のこと

ネクストホップ　…　宛先に向かうために経由する隣接している次のルータのこと

ARPテーブル　…　ＡＲＰのやり取りを登録している（ＩＰアドレスとＭＡＣアドレスの関係情報）

ARP　…ＩＰアドレスからＭＡＣアドレスを調べる仕組みのこと

コアルータ　…　基幹ネットワークのバックボーンを構成するルータで、高速なパケット転送を実現する。大型のルータが適用される
　　　　　　　　→通信キャリアやＩＳＰネットワークの通信を中継するために利用

センタールータ　企業の拠点間ＷＡＮネットワークで本社拠点やデータセンタなど、企業の中心核にてネットワークを接続するための適用される

エッジルータ　…　センタールータよりちょっと低め。支社や営業所にて利用される

ブロードバンドルータ…　家庭環境やＳＯＨＯ環境でインターネット回線に接続する目的で利用される

 

ルーティング　…目的の場所あｍでルートを選択すること

直接接続　…ルータが直接接続しているネットワークのルート情報

スタティックルーティング　…管理者が宛先ネットワークへの最適なルートを「手動」で静的に設定したルートを使用したルーティング
ダイナミックルーティング　…ルータで設定されたルーティングプロトコルで「自動」で動的に追加されるルートを使用したルーティング

ディスタンスベクタ型　…隣接ルータ動詞で、経路情報そのものを変換しあい、宛先までのホップ数が最短になる経路
リンクステート型　…隣接関係（ネイバー）を確立したルータから接続状況の情報を交換しあい、この情報を再構成して全体の接続状況を再構築する
　　　　　　　　　　全体の接続状況をものに最適な経路を計算してルーティングテーブルを作成する」　
ハイブリッド型…（拡張ディスタンスベクタ型）リンクステート型とディスタンスベクタ型の利点をあわせた
 

■ トランスポート層　…通信を制限・制御する層

コネクション型通信　…　データの送信をする前に相手と通信できるか確認を得たうえで送信する方法（コネクション確立してから）

ACK　…相手からの要求を承認して応答すること

コネクションレス型通信　…相手の回答を得ずに、いきなりデータ送信する方法（データグラム方式）

TCP　…　コネクション型の代表例

スリーウェイハンドシェイク　…ＳＹＮ→ＡＣＫ→ＳＹＫ＆ＡＣＫ　の回答を得て初めてデータが送信される

Telnet　…暗号化されていない情報をリモートで接続する。遠隔地にあるサーバーやルータなどを遠隔操作するときに使用するプロトコル

HTTP　…　ハイパーテキスト転送のルール　（ポート番号８０）

FTP　…　ファイル転送のルール（ポート番号２１）

SMTP　…　簡易版メール転送の通信ルール。メールを送信する（ポート番号２５）

TCPヘッダ　…　アプリケーション層から渡されたデータにＴＣＰヘッダを付加して、ＴＣＰセグメントとしてネットワーク層に渡す

UDP　…　TCPに比べ、早くデータを転送するタイプ。コネクションレス型。ベストエフォートな通信

RTP　…音声や映像のデジタルデータによって構成されたＩＰパケットで受信側で音声や映像の再生を実現させるためにＵＤＰの上位層で動作するプロトコル

SNMP　…　ネットワーク上の機器を監視、制御するためのプロトコル。監視を行っていれば即座に障害に対応できる

DHCP　…　コンピュータがネットワークに接続する際に必要な設定情報を自動的に割り当てる接続をＩＰアドレスに自動的に割り当てること

TFTP　…　簡易的なファイル転送プロトコル。ユーザ認証を行わず、ファイル転送する。ルータの設定ファイルなどサイズがあまり大きくないファイルを簡単に転送したいという用途で使われる

RIP　…　ＡＳ内部における経路情報を交換するために用いられる。自動的にアップロードするという役割

ＵＤＰへッダ　…アプリケーションそうから渡されたデータにＵＤＰデータを付加し、
　　　　　　ＵＤＰデータグラムとしてネットワーク層に渡すデータのこと（＝このデータを、ＵＤＰペイロードという）
